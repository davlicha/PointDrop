# Data Dictionary: Лояльність (PointDrop) 

---

### 1. Сутність: `users` (Користувачі)
**Опис:** Зберігає облікові дані як звичайних клієнтів, так і адміністраторів закладів.

| Назва поля | Тип даних | Обмеження | Опис | Приклад |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | PK, Not Null | Унікальний ідентифікатор | 550e8400-e29b... |
| `phone` | VARCHAR(20) | Unique, Not Null | Номер телефону для авторизації | +380501234567 |
| `email` | VARCHAR(255) | Unique, Nullable | Опціональна пошта | user@mail.com |
| `password_hash` | VARCHAR(255) | Not Null | Хеш пароля (BCrypt) | $2b$10$xyz... |
| `name` | VARCHAR(100) | Not Null | Ім'я клієнта або власника | Іван Франко |
| `role` | ENUM | Not Null, Default: CUSTOMER | Роль доступу (CUSTOMER, ADMIN) | CUSTOMER |
| `created_at` | TIMESTAMP | Not Null, Default: Now | Час реєстрації | 2026-04-10 12:00 |

---

### 2. Сутність: `merchants` (Заклади)
**Опис:** Бізнес-сутність, яка керує власною програмою лояльності та балами.

| Назва поля | Тип даних | Обмеження | Опис | Приклад |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | PK, Not Null | Ідентифікатор закладу | a1b2c3d4-e5f6... |
| `name` | VARCHAR(255) | Not Null | Назва бізнесу (кав'ярні/ресторану) | Lviv Croissants |
| `cashback_rate` | FLOAT | Not Null | Відсоток нарахування балів | 10.5 |
| `admin_id` | UUID | FK, Not Null | Посилання на власника (Table: `users`) | 550e8400-e29b... |

**Стратегія поведінки (FK Strategy):**
* `admin_id`: **ON DELETE RESTRICT**. Системно заборонено видаляти акаунт адміністратора, якщо до нього прив'язаний існуючий заклад, щоб не втратити управління бізнесом.

---

### 3. Сутність: `balances` (Баланси)
**Опис:** Віртуальний рахунок, що зв'язує конкретного користувача із конкретним закладом.

| Назва поля | Тип даних | Обмеження | Опис | Приклад |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | PK, Not Null | ID запису балансу | c8f9g0h1-j2k3... |
| `user_id` | UUID | FK, Not Null | Чий це баланс | 550e8400-e29b... |
| `merchant_id` | UUID | FK, Not Null | До якого закладу належить | a1b2c3d4-e5f6... |
| `points_amount` | INT | Not Null, Default: 0 | Поточна кількість накопичених балів | 150 |
| `qr_payload` | VARCHAR(255) | Unique, Nullable | Зашифрований рядок для ідентифікації | eyJhbGciOi... |

**Обмеження цілісності:**
* `UNIQUE(user_id, merchant_id)`: Один юзер не може мати два різних баланси в межах одного закладу.

**Стратегія поведінки (FK Strategy):**
* `user_id`: **ON DELETE CASCADE**. При видаленні профілю клієнта всі його баланси видаляються.
* `merchant_id`: **ON DELETE CASCADE**. При закритті (видаленні) закладу всі баланси клієнтів цього закладу знищуються.

---

### 4. Сутність: `transactions` (Транзакції)
**Опис:** Журнал аудиту (фінансовий лог). Записує нарахування (EARN), списання (REDEEM) та перекази (TRANSFER / PointDrop).

| Назва поля | Тип даних | Обмеження | Опис | Приклад |
| :--- | :--- | :--- | :--- | :--- |
| `id` | UUID | PK, Not Null | ID транзакції | t1r2a3n4-s5a6... |
| `type` | ENUM | Not Null | Тип (EARN, REDEEM, TRANSFER) | EARN |
| `amount` | INT | Not Null | Кількість балів | 50 |
| `merchant_id` | UUID | FK, Not Null | Де саме відбулася дія | a1b2c3d4-e5f6... |
| `sender_id` | UUID | FK, Nullable | Відправник (NULL якщо це EARN) | 550e8400-e29b... |
| `receiver_id` | UUID | FK, Not Null | Отримувач балів | d4c3b2a1-f6e5... |
| `timestamp` | TIMESTAMP | Not Null, Default: Now | Точний час операції | 2026-04-10 12:05 |

**Стратегія поведінки (FK Strategy):** *Тут застосовано ACID-підхід: фінансова історія не повинна видалятися.*
* `merchant_id`: **ON DELETE RESTRICT**. Не можна видалити заклад, якщо за ним є фінансова історія.
* `receiver_id`: **ON DELETE RESTRICT**. Не можна видалити акаунт клієнта, якщо він отримував бали (аудит).
* `sender_id`: **ON DELETE SET NULL**. Якщо клієнт видаляє свій акаунт, історія переказу (PointDrop) залишається у отримувача, але відправник стає анон
* імним (`NULL`).
